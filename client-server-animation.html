Request-Response Cycle

1
User Interaction
User clicks a button or submits a form in the browser
2
Client-Side JavaScript Executes
Browser JavaScript captures the event and sends an HTTP request (GET, POST, etc.) to the server
3
Request Travels to Server
The HTTP request travels over the internet to reach the server
4
Server-Side JavaScript Processes
Node.js receives the request, runs business logic, queries database if needed
5
Server Sends Response
Server sends back JSON data, HTML, or other response to the client
6
Client Updates UI
Browser JavaScript receives response and updates the page dynamically

Client-Side JavaScript

  • Runs in the browser
  • Manipulates DOM elements
  • Handles user interactions
  • Makes API requests
  • Validates form inputs
  • Creates animations

Server-Side JavaScript

  • Runs on server (Node.js)
  • Processes HTTP requests
  • Manages databases
  • Implements business logic
  • Authentication & security
  • Sends responses to clients

Why JavaScript for Both?

  • One language for full stack
  • Code reuse between client/server
  • Large ecosystem (npm)
  • Fast development cycle
  • JSON native support
  • Active community
